# Google Workspace (Gmail, Calendar, Drive)

Use this for reading/sending Gmail, managing Google Calendar events, and working with Google Drive (Docs and files). The tools operate on the user's authenticated Google account.

## When to use

Use Google Workspace tools when the user asks to:
- Read, search, or send email (Gmail)
- List, create, update, or delete calendar events
- Search Drive, create/update Google Docs, or manage files (delete, move to bin)

## Available tools (high level)

### Gmail

#### `google_workspace__read_gmail_emails`
Reads the latest N emails from the user's Gmail inbox (N = `num_emails`). Excludes auto-generated Gmail messages (e.g. "This message was automatically generated by Gmail.") and messages without a plain text or HTML body. Returns a markdown string with id, snippet, and body for each.

**Args:** `num_emails` (int) ‚Äì Number of emails to read.

**Best for:** Reading a configurable number of recent emails when you don't need exactly 10 or a subject search.

#### `google_workspace__get_last_10_emails`
Gets the last 10 emails from the user's Gmail inbox. Excludes auto-generated Gmail messages. Returns id, snippet, and body for each (markdown).

**Best for:** Scanning recent inbox activity or catching up on the latest messages without searching by subject.

#### `google_workspace__read_email_by_subject`
Searches for emails by subject and returns the body and snippet of the most recent matches (up to 5).

**Args:** `subject` (string) ‚Äì The subject line to search for.

**Best for:** Finding a specific thread or recent emails about a topic.

#### `google_workspace__send_email`
Sends an email from the user's Gmail account.

**Args:** `email_content` ‚Äì Structured object with `to`, `subject`, and `body` (not `recipient`).

**Best for:** Composing and sending a single email with clear recipient/subject/body.

#### `google_workspace__mark_email_as_read`
Marks an email as read by removing the UNREAD label.

**Args:** `message_id` (string) ‚Äì The Gmail message ID (from `read_gmail_emails`, `get_last_10_emails`, or `read_email_by_subject`).

**Best for:** Clearing the unread state of a specific email after the user has dealt with it.

### Google Calendar

#### `google_workspace__list_calendar_events`
Lists calendar events in a time range, optionally filtered by a text query.

**Args:**
- `start_time` (string) ‚Äì Start of range in ISO 8601 (e.g. `2025-07-05T00:00:00Z`).
- `end_time` (string) ‚Äì End of range in ISO 8601 (e.g. `2025-07-06T00:00:00Z`).
- `query` (optional) ‚Äì Text filter (e.g. `meeting`).

**Best for:** Seeing what‚Äôs on the calendar in a given window or finding events by keyword.

#### `google_workspace__create_calendar_event`
Creates a new calendar event from structured details.

**Args:** `event_details` ‚Äì Object with `summary`, `start_time`, `end_time` (ISO 8601), and optional `description`.

**Best for:** Scheduling a new event with a clear title and time.

#### `google_workspace__update_calendar_event`
Updates an existing event by ID; only provided fields are changed.

**Args:**
- `event_id` (string) ‚Äì ID of the event to update.
- `update_details` ‚Äì Object with optional fields to change: `summary`, `start_time`, `end_time`, `description`.

**Best for:** Changing time, title, or description of an existing event without resending unchanged fields.

#### `google_workspace__delete_calendar_event`
Deletes a calendar event by ID. Get the ID from `list_calendar_events` or a search.

**Args:** `event_id` (string) ‚Äì Unique ID of the event to delete.

**Best for:** Cancelling or removing a specific event.

### Google Drive

#### `google_workspace__list_drive_files`
Searches Drive with a query string.

**Args:** `query` (string) ‚Äì Search query. Examples: `name contains 'report'`, `mimeType='application/vnd.google-apps.spreadsheet'`. See Google Drive API docs for full query syntax.

**Best for:** Finding files by name, type, or other Drive metadata.

#### `google_workspace__create_drive_document`
Creates a new Google Document with the given title and optional initial content.

**Args:**
- `title` (string) ‚Äì Title of the new document.
- `content` (optional) ‚Äì Initial text content.

**Best for:** Creating a new Doc with or without starter text.

#### `google_workspace__update_drive_document`
Overwrites the content of an existing Google Document.

**Args:**
- `file_id` (string) ‚Äì ID of the document to update.
- `content` (string) ‚Äì New full text content.

**Best for:** Replacing the entire body of a Doc (overwrite, not append).

#### `google_workspace__delete_drive_file`
Permanently deletes a file from Drive. **Cannot be undone.**

**Args:** `file_id` (string) ‚Äì ID of the file to delete.

**Best for:** Permanent removal when the user explicitly wants the file gone.

#### `google_workspace__move_drive_file_to_bin`
Moves a file to the Drive bin (trash). It can be restored later.

**Args:** `file_id` (string) ‚Äì ID of the file to move to the bin.

**Best for:** Soft delete when the user might want to restore the file.

## Choosing the right tool

- **Read N recent emails ‚Üí** `google_workspace__read_gmail_emails(num_emails)`.
- **Last 10 emails ‚Üí** `google_workspace__get_last_10_emails`.
- **Find email by topic ‚Üí** `google_workspace__read_email_by_subject(subject: "...")`.
- **Send email ‚Üí** `google_workspace__send_email(email_content: { to, subject, body })`.
- **Mark email read ‚Üí** `google_workspace__mark_email_as_read(message_id)`.
- **See calendar in a range ‚Üí** `google_workspace__list_calendar_events(start_time, end_time)`; add `query` to filter.
- **Create event ‚Üí** `google_workspace__create_calendar_event(event_details)` with `summary`, `start_time`, `end_time`, optional `description`.
- **Change event ‚Üí** `google_workspace__update_calendar_event(event_id, update_details)` (partial fields).
- **Remove event ‚Üí** `google_workspace__delete_calendar_event(event_id)`.
- **Find Drive files ‚Üí** `google_workspace__list_drive_files(query)`.
- **New Doc ‚Üí** `google_workspace__create_drive_document(title, content?)`.
- **Replace Doc content ‚Üí** `google_workspace__update_drive_document(file_id, content)`.
- **Soft delete file ‚Üí** `google_workspace__move_drive_file_to_bin(file_id)`.
- **Hard delete file ‚Üí** `google_workspace__delete_drive_file(file_id)` (irreversible).

## Example user requests

- "What‚Äôs my latest email?" ‚Üí `read_gmail_emails(num_emails)` (e.g. 1 or 3). "Read my last 3 emails" ‚Üí same.
- "Show me my last 10 emails" / "What's in my inbox lately?" ‚Üí `get_last_10_emails`.
- "Find emails about 'invoice'" ‚Üí `read_email_by_subject("invoice")`.
- "Send an email to alice@example.com with subject 'Meeting' and body 'See you at 3.'" ‚Üí `send_email` with `email_content: { to: "alice@example.com", subject, body }`.
- "What‚Äôs on my calendar tomorrow?" ‚Üí `list_calendar_events` with tomorrow‚Äôs start/end in ISO 8601.
- "Create a meeting 'Standup' next Monday 9‚Äì9:30am." ‚Üí `create_calendar_event` with summary, start_time, end_time.
- "Delete the event with ID xyz." ‚Üí `delete_calendar_event("xyz")`.
- "Find my spreadsheets in Drive." ‚Üí `list_drive_files("mimeType='application/vnd.google-apps.spreadsheet'")`.
- "Create a Doc called 'Notes' with 'Hello world'." ‚Üí `create_drive_document("Notes", "Hello world")`.
- "Overwrite Doc abc123 with this new text." ‚Üí `update_drive_document("abc123", "new text")`.
- "Move file xyz to trash." ‚Üí `move_drive_file_to_bin("xyz")`.

## Safety and performance notes

- **Destructive actions:** `delete_drive_file` is permanent; prefer `move_drive_file_to_bin` when the user might want to restore. Confirm before permanent delete when possible.
- **Calendar event IDs:** For update/delete, you need the event ID; get it from `list_calendar_events` first.
- **Gmail message IDs:** For `mark_email_as_read`, use the message ID returned by `read_gmail_emails`, `get_last_10_emails`, or `read_email_by_subject`.
- **Drive file IDs:** For update/delete/move, use the file ID returned by `list_drive_files` or `create_drive_document`.
- **Times:** Use ISO 8601 for calendar start/end (e.g. `2025-07-05T00:00:00Z`). Respect the user‚Äôs timezone when interpreting "tomorrow" or "next Monday."
- **Credentials:** If tools fail with "credentials not available or invalid", the user must run `python get_credentials.py` in the MCP server directory to re-authenticate.
